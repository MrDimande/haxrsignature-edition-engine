## 🧩 System Design

### 🔹 Single Engine Principle

One system renders all invitations dynamically:

```
/[slug]
```

### 🔹 Example Routes

- `/jessicakulaya`
- `/lobolo-jessica-samuel`
- `/traditional-wedding`

---

## 🔐 Security Model

- Root route does NOT expose invitations
- No public listing of events
- All access is slug-based
- Invalid routes return 404
